## Entornos virtuales

Los entornos virtuales sirver para instalar las dependencias de un proyecto sin afectar al sistema global.

### uv

Mi propuesta es usar `uv`.

- Es parecido a trabajar con node. Tienes un fichero pyproject.toml donde defines las dependencias y luego con `uv sync` se instalan.
- Para instalar una nueva dependencia, se hace con `uv add <package>` y se actualiza el pyproject.toml automáticamente. Esto es más optimo que un requirements.txt porque no tienes que añadirla manualmente.
- A demás se estipula la versión de python a usar, lo que es importante para la reproducibilidad del proyecto.
- Todo el equipo trabajamos con el mismo entorno virtual, lo que evita problemas de compatibilidad entre diferentes versiones de paquetes o de python.
- A demás, `uv` facilita la creación de librerías y su publicación en PyPI, lo que es útil para nuestro proyecto.

### Ejemplo de uso de `uv`:

Iniciar repositorio y crear entorno virtual:

```bash
uv init
uv add slither
uv add mythril
```

Desccargar repositorio y sincronizar entorno virtual:

```bash
git clone https://github.com/pardalaco/TFM-UNIR.git
cd TFM-UNIR
uv sync

myth version
slither --version
```

Iniciar repositorio y crear librería:

```bash
uv init --lib mylib
```
